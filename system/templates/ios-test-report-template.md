<!-- Referenced by: ../pipelines/ios-testing-pipeline.md -->

## Usage Notes

1. Replace all `{placeholder}` values with actual test data
2. Add or remove flow sections as needed based on test scope
3. Ensure every issue has a corresponding screenshot reference
4. The Google Docs link is optional (depends on pipeline configuration)
5. Maintain the issue ID format (H-001, M-001, L-001) for traceability
