## Contents

### `prompts/`

Contains all agent prompt templates used by the orchestrator when spawning subagents. Each template defines the complete context, instructions, and constraints for a specific agent role.

**Template Categories:**

| Category | Templates | Purpose |
|----------|-----------|---------|
| **Tech-Specific** | `kotlin-spring-boot-backend-developer.md`, `react-native-expo-frontend-developer.md`, `kotlin-spring-postgres-architect.md` | Role-specific development agents |
| **Quality** | `qa-tester.md`, `kotlin-spring-react-native-judge.md` | Testing and review agents |
| **Design** | `ios-ux-designer.md` | UX/UI design agents |
| **Product** | `fitness-product-ideator.md`, `flashcards-product-ideator.md`, `spend-manager-product-ideator.md` | Product ideation agents |
| **Operations** | `ci-agent.md`, `infrastructure-engineer.md`, `documentation.md`, `retrospective.md` | DevOps and documentation agents |
| **Utilities** | `google-docs-writer.md` | Tool-specific utility agents |

**How to Use:**

1. Select the appropriate template for the agent role
2. Copy the template content
3. Replace template variables (e.g., `{spawned_by}`, `{what to implement}`)
4. Replace path placeholders (e.g., `{backend-folder}`, `{frontend-folder}`) with project-specific values
5. Spawn the agent with the filled template

**Session Logs:** All templates reference `system/formats/session-log.md` for consistent log formatting.

**Variable Reference:** See [`prompts/README.md`](prompts/README.md) for the complete variable definitions, examples, and relevance rules for selecting knowledge files.

### `ios-test-report-template.md`

A standardized template for iOS testing reports generated by the iOS Testing Pipeline. Provides consistent formatting for:

- Test environment documentation
- Test results by flow
- Issue tracking with severity levels (HIGH/MEDIUM/LOW)
- Visual/UX assessment sections
- Recommendations and execution logs

**Referenced by:** `../pipelines/ios-testing-pipeline.md`

## Adding New Templates

When adding new templates:

1. **Prompt templates**: Add to `prompts/` and update `prompts/README.md` with variable definitions
2. **Report templates**: Add to root `templates/` directory with clear naming convention
3. **Update this README**: Document the new template in the appropriate section
